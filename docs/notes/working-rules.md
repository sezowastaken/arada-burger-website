# Working Rules

To keep the Arada Burger project manageable and high-quality, follow these rules:

## 1. Keep it Simple
- If there is a simple way and a "clever" way, choose the simple way.
- Avoid unnecessary libraries. Use native features whenever practical.

## 2. Build for the Current Phase
- Focus on the digital menu first. Don't build the database schema for the admin panel until we reach Phase 2.

## 3. Understandable Code
- Write code that a junior developer could read and understand.
- Use clear variable names and follow established project patterns.

## 4. Practical Infrastructure
- Docker is a tool for deployment and basic local testing.
- Don't let Docker configuration complicate the daily development workflow.

## 5. Documentation First
- Update `asset-inventory.md` when adding new images.
- Keep `README.md` and `roadmap.md` up to date with progress.

## 6. Frontend Specifics
- Focus on mobile responsiveness first (most customers view menus on phones).
- Ensure bilingual support is handled cleanly from the start.
